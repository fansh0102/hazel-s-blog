概述

在 Kubernetes 集群中实现高效的 GPU 调度是智算平台的核心能力。本页介绍我们在 GPU 调度方面的实践经验，包括设备插件配置、拓扑感知、GPU 共享和多租户隔离等技术。

NVIDIA Device Plugin

Kubernetes 通过 NVIDIA Device Plugin 来发现和管理 GPU 资源。部署方式如下：

# 部署 Device Plugin
kubectl apply -f https://raw.githubusercontent.com/NVIDIA/k8s-device-plugin/master/deployments/static/static.yaml

# 验证部署
kubectl get pods -n kube-system | grep nvidia-device-plugin

Pod 中申请 GPU 资源：

resources:
  limits:
    nvidia.com/gpu: 1

拓扑感知调度

对于分布式训练场景，GPU 之间的拓扑关系（NVLink、PCIe）严重影响通信效率。Kubernetes Topology Manager 可以帮助优化 Pod 调度。

# 开启 Topology Manager
kubelet --topology-manager-policy=best-effort

# 在 Pod 中配置拓扑策略
topologyManagerPolicy: best-effort
topologyManagerScope: container

GPU 共享策略

对于小 workload，可以采用 GPU 共享策略提高资源利用率：

  • Time Slicing：将 GPU 时间片分给多个任务
  • MIG（Multi-Instance GPU）：A100/H100 硬件级切分
# 配置 Time Slicing
config.yaml:
  version: v1
  sharing:
    timeSlicing:
      resources:
        - name: nvidia.com/gpu
          replicas: 4

多租户隔离

在共享集群中，需要做好租户间的资源隔离：

  • 使用 ResourceQuota 限制每个 namespace 的 GPU 配额
  • 配置 LimitRange 设定默认 GPU 申请
  • 结合 PriorityClass 实现抢占式调度
apiVersion: v1
kind: ResourceQuota
metadata:
  name: gpu-quota
spec:
  hard:
    requests.nvidia.com/gpu: 10
